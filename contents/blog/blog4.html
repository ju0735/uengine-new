[데이터메쉬] 기관/시스템 간 데이터를 공유하는 3가지 방법

Date: April 04, 2024

 [데이터메쉬] 기관/시스템 간 데이터를 공유하는 3가지 방법

  1. 캐노니컬 모델을 만들어서 (기준 정보를 통일) 연동되게 하는
    (싱글모델 - 모놀리쓰)
  2. 빅데이터 플랫폼을 만들어서 ETL을 "통합적"으로 수행
    (여전히 싱글모델 - 큰 문제영역)
  3. 데이터 메시로 도메인 별로 필요한 ETL을 "한정적"으로 수행
    (소비자 중심의 멀티모델, 한정된 문제영역)

1>2>3 순으로 통합/유지보수 비용이 높고, 유연성이 떨어짐

3의 전략 (데이터 메시)를 수행하는 효과적인 방법으로 스트림 프로세싱을 통하여 구축하는 기법이 있음.

Data Mesh Patterns: Event Streaming Backbone

관련 솔루션

https://datavault-builder.com/automate-your-data-mesh-implementation/

LLM을 응용하면 데이터 기준 정보를 (스키마 매핑) 자동으로 매핑해줄 수 있는 자동화도 가능

https://medium.com/@shrinath.suresh/ai-powered-schema-mapping-95f596d31590
